CADASTRO: 
Textualmente: 
Temos o conceito de que a aplicação atua tanto a nível físico quanto a nivel jurídico, entregando
ao usuário final o avanço da economia circular dentro do contexto do site /aplicativo, visando trazer 
transparência sobre como a somos afetados por empresas (positivamente ou negativamente), assim como trazer aos 
usuários físicos a possibilidade de participar em seus projetos. 


Desenvolvimento:
Na prática para atuar dessa forma, é preciso modelar um digrama para o fluxo de cadastro de pessoas
(tanto física quanto jurídica), esse cadastro precisa de Alguns parametros fundamentais para passar
 para o banco de Dados, tais Como:

	Id: Id seria gerado automaticamente, único para cada registro 
	Nome: Todo usuário necessita de Uma Descrição. Como ser chamado? esse Campo permitiria um texto de 100 Caracteres
	Tipo: o tipo se refere justamente ao tipo de Pessoa, se seira um pessoa física ou jurídica
	Senha:Senha do usuário para realizar login
	Login: Texto para fazer login em sua conta
	Residencia: Localidade padrão, onde reside o usuário ou empresa cadastrada

E também teriamos um parametro próprio, como:
Pontuação: por Padrão teria valor 0, indicando uma posição neutra. Seria um valor gerado pelo sistema, indicando que este usuário as ações (positivas ou negativas), resultando em + ou menos pontos (Inicial 0 Pontos)
